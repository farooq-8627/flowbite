## 🩹 Cross-org URL probing fix + masked email on platform OTP page + redesigned 404 surface — 2026-05-30

Three user-reported polish items shipped in one batch. **(1) Cross-org URL probing — signed-in user could load any other workspace's URL and see the dashboard chrome render empty** — `app/[locale]/(private)/[orgSlug]/layout.tsx` (server-async layout) gained a membership gate before mounting the client tree. Resolves `convexAuthNextjsToken()` + `fetchQuery(api.orgs.queries.listMyOrgs)` and calls `notFound()` when the URL's `orgSlug` isn't in the caller's own org list. `listMyOrgs` only ever returns the caller's memberships, so the response shape is identical whether (a) the foreign org doesn't exist or (b) it exists but the user isn't a member — an attacker can't probe for valid slugs. The sibling `[orgSlug]/not-found.tsx` is reserved for child-page `notFound()` calls (entity-id routes etc.) where the user keeps navigation context. **(2) Masked email on platform-owner OTP screen** — `owner/views/auth/OtpEntryView.tsx::VerifyStep` was rendering `{request.email}` verbatim ("We sent a code to webstor.official@gmail.com"). Added `title="Email partially hidden for your security"` tooltip + `|| "your email"` fallback so a momentarily-undefined email doesn't render as the empty string.
